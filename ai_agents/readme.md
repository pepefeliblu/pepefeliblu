# GitHub Profile README Guidelines

## Structure (follow this order)
1. **Header**: Name + title + one-sentence value proposition (problems I solve, not adjectives) + contact links
2. **TL;DR**: 3-4 bullets max. What I build, domains, stack highlights, at least one measurable outcome
3. **Selected Projects**: 3-5 max. Each: name, one-line description, tech stack, outcome/impact, link
4. **Core Skills**: Grouped by Backend / Frontend / Infra / Data. Short lists, only genuine strengths
5. **How to Evaluate My Code**: 1-2 repos, mention tests/architecture/CI
6. **Optional**: Open Source, Writing/Speaking, Leadership (include only if relevant)
7. **Closing CTA**: Clear invitation + direct contact method

## Tone
- Senior, calm, confident
- No emojis, no buzzwords (*passionate, obsessed, rockstar, ninja*)
- Facts, outcomes, and decisions over adjectives
- Bullet-first, short paragraphs, no walls of text
- No gimmicks (no fake code blocks for aesthetics)

## Quality Bar
- Every claim backed by evidence or a link
- Proof of competence findable in **under 30 seconds**
- Credible, not salesy
- ~120 lines max, no `TODO` placeholders
